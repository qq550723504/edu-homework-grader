# Security Policy

本项目处理未成年人教育数据。请不要在公开 Issue 中提交真实学生姓名、账号、作业内容、成绩、令牌或数据库导出。

发现安全问题时，应通过私有渠道联系仓库维护者。首期安全基线包括：最小化收集、租户隔离、最小权限、全链路 HTTPS、审计日志、密钥不入库、评分输入白名单和批改进程资源限制。

## 试点发布检查

- 生产环境使用至少 32 字节的 `AUDIT_HMAC_KEY`，记录其版本并按学校密钥轮换流程更新。
- 审核 `PROCESSOR_ALLOWED_HOSTS`；任何新模型或文本处理器先进入 [数据清单](docs/data-inventory.md)。
- 每日将每个租户的审计链头导出到权限隔离的独立存储，并保留 Keycloak 登录事件。
- 普通应用日志不得用于查看学生作答；日志中禁止记录答案、令牌、学校编号、OIDC subject、姓名、邮箱或连接字符串。
- 对未满 14 周岁的学生，由学校在平台外核验监护人授权并按学校制度保留原始证据；平台只接收不含联系人或证件信息的凭证引用、通知版本和同意状态。
- 导入前检查花名册同意状态；授权时使用管理员接口记录通知版本和非个人凭证引用。收到撤回后立即调用撤回接口，阻止新的作业、评分和申诉处理。
- 撤回后现有记录只可为法定保存和安全目的留存；导出、删除和保全处置须走独立审批流程。
- 数据删除由租户管理员在平台外完成学校核验后登记；如存在法定保存、安全事件或争议，应先设置保全状态，不能批准或清理。
- 批准请求时设定适用的最早清理时间；先运行 `python -m edu_grader_api.privacy_cleanup` 审阅候选项。仅在复核无保全后，使用 `--execute --request-id <UUID> --actor-user-id <admin UUID>` 一次处理一个请求。
- 不得用直接 SQL 删除学生记录。受控清理会去标识化身份并保留不可变审计关联；审计账本继续按三年规则保存。
