# Security Policy

本项目处理未成年人教育数据。请不要在公开 Issue 中提交真实学生姓名、账号、作业内容、成绩、令牌或数据库导出。

发现安全问题时，应通过私有渠道联系仓库维护者。首期安全基线包括：最小化收集、租户隔离、最小权限、全链路 HTTPS、审计日志、密钥不入库、评分输入白名单和批改进程资源限制。

## 试点发布检查

- 生产环境使用至少 32 字节的 `AUDIT_HMAC_KEY`，记录其版本并按学校密钥轮换流程更新。
- 审核 `PROCESSOR_ALLOWED_HOSTS`；任何新模型或文本处理器先进入 [数据清单](docs/data-inventory.md)。
- 每日将每个租户的审计链头导出到权限隔离的独立存储，并保留 Keycloak 登录事件。
- 普通应用日志不得用于查看学生作答；日志中禁止记录答案、令牌、学校编号、OIDC subject、姓名、邮箱或连接字符串。
