# 自适应学习试点实验协议

状态：Draft  
阶段：教师影子模式与有限学生试点  
适用范围：G4–G9 数学 M1/M2、英语 E1/E2；E3/E4 仅使用教师最终确认结果

## 1. 目的

本协议用于验证学生知识画像和自适应练习是否：

- 产生可解释、稳定且教师认可的知识状态；
- 改善学习增益、延迟保持和迁移表现；
- 不增加连续失败、挫败或不公平学习机会；
- 不影响正式成绩；
- 不把未经审核的 AI 题展示给学生；
- 在同意、隐私和安全门禁下可运行、可暂停、可回滚。

本试点不是正式考试，也不用于证明学生的固定能力。

## 2. 核心假设

### H1：画像准确性

教师认为“明显错误”的知识状态比例低于约定门槛，且低证据学生不会被错误标记为已掌握。

### H2：教师价值

教师推荐计划的直接接受率达到初始目标，修改后接受率进一步提高。

### H3：学习效果

接受自适应练习的学生在目标知识点上的后测、延迟保持或迁移表现不低于对照/基线，并在至少一个预先声明的主要指标上改善。

### H4：体验安全

自适应组的连续失败率、放弃率、提示依赖和异常练习时长不明显劣于对照/基线。

### H5：机会公平

不同年级、课程目标、设备/网络条件和证据量群体获得的难度、探索机会和课程覆盖不存在无法解释的系统性差异。

## 3. 非目标

本协议不评估：

- 正式成绩自动化；
- 分班或升留级；
- K 阶段和 G13；
- 几何证明、开放性建模或英语作文自动判定；
- 未经教师审核的实时 AI 出题；
- 长期职业或升学预测；
- 将点击率、做题量或停留时长作为唯一成功指标。

## 4. 试点阶段

### Stage 0：离线回放

- 不影响线上用户；
- 使用历史或合成 Learning Events；
- 运行 Student Model v1；
- 检查状态稳定性、低证据误标、教师改分替代和删除流程；
- 生成分层评估报告。

退出条件：事件重放稳定、数据泄漏检查通过、异常状态跳变在门槛内。

### Stage 1：教师影子画像

- 知识状态只对授权教师可见；
- 不向学生展示；
- 不自动推荐；
- 教师查看证据、确认或覆盖状态；
- 收集明显错误、证据不足和错误模式反馈。

退出条件：教师覆盖率和明显错误率达到门槛，隐私影响评估获批。

### Stage 2：教师推荐助手

- 系统生成练习计划建议；
- 只有教师确认后才布置；
- 记录接受、修改、拒绝和原因；
- 推荐只使用已发布题目；
- 不影响正式成绩。

退出条件：教师接受率、内容质量、连续失败模拟和题库覆盖达到门槛。

### Stage 3：有限学生自适应

- 按租户、班级、年级、学科和目标启用 Feature Flag；
- 学生完成低风险课后练习；
- 提供退出、暂停和人工帮助；
- 有明确的对照或准实验设计；
- 实时监控体验和安全指标。

### Stage 4：扩大或停止

仅在预先声明的学习效果、安全、公平性和运维门槛通过后扩大。未通过时停止、回滚并执行数据清理或保留决定。

## 5. 参与者与范围

### 5.1 参与标准

- 处于已批准的 G4–G9 班级；
- 课程 profile 和目标映射已审核；
- 学生处理权限与监护人同意状态允许；
- 教师完成试点培训；
- 设备和网络满足最低要求；
- 学校、产品、教学、运维和合规批准。

### 5.2 排除标准

- 存在处理限制或同意撤回；
- 正在进行正式高风险考试；
- 课程目标或题目映射未审核；
- 需要无障碍支持但当前版本尚不能满足；
- 班级正在参与可能干扰结果的冲突实验；
- 运维监控或 Kill Switch 不可用。

### 5.3 样本量

样本量不能凭方便程度固定。实施前根据：

- 主要学习效果指标；
- 预期最小有意义差异；
- 班级内相关性；
- 目标知识点覆盖；
- 缺失和退出比例；
- 可用班级数量；
- 影子阶段观察到的方差；

完成统计功效分析并记录。小样本阶段只用于可用性和安全性，不宣称学习效果结论。

## 6. 实验设计

优先顺序：

1. 班级或学生层面的集群随机对照；
2. 交叉设计；
3. 分阶段 rollout；
4. 匹配对照或差分中的差分等准实验；
5. 只有在无法建立对照时，使用前后测并明确局限。

随机化单位需要避免学生之间共享题目造成污染。

建议分组：

- Control：教师按现有方式布置已审核练习；
- Treatment A：教师看到画像和推荐，但手动确认；
- Treatment B：低风险自适应练习，仅在 Stage 3 启用。

正式成绩、授课内容和教师基本支持应保持一致。

## 7. 主要结果指标

试点启动前选择一项或两项主要指标并锁定分析计划。

可选主要指标：

- 目标知识点后测提升；
- 2–4 周延迟保持；
- 未见过的迁移题表现；
- 达到教师定义掌握门槛所需练习题数。

不得在看到结果后随意更换主要指标。

## 8. 次要指标

### 模型质量

- 低证据误标掌握率；
- 教师认为画像明显错误的比例；
- 状态跳变率；
- 概率校准；
- 状态重放一致性；
- 教师覆盖率和原因。

### 推荐质量

- 教师直接接受率；
- 修改后接受率；
- 拒绝原因；
- 题目重复率；
- 难度命中率；
- 先修题命中率；
- 推荐 API 延迟。

### 学习体验

- 连续失败率；
- 练习主动退出率；
- 提示依赖；
- 单次练习时长；
- 断网恢复成功率；
- 学生和教师满意度；
- 人工帮助请求率。

### 运维与安全

- Learning Event 延迟和积压；
- 状态更新失败率；
- 跨租户拒绝；
- 数据删除遗漏；
- Grader 降级率；
- 推荐 Kill Switch 使用次数；
- 隐私或安全事件。

## 9. 公平性指标

至少按以下维度分层：

- 年级；
- 学科和题型；
- 课程 profile；
- 证据量；
- 设备类型；
- 网络质量；
- 无障碍需求；
- 教师/班级。

检查：

- 推荐难度分布；
- 探索题比例；
- 提示机会；
- 课程目标覆盖；
- 连续失败；
- 学习增益；
- 退出率；
- 教师覆盖率。

受保护属性仅可在获批的离线公平性审计中使用，与在线推荐隔离，并只输出聚合结果。

## 10. 题目与内容控制

- 自适应练习只能选择已验证、教师审核并发布的题目；
- 题目必须关联审核过的课程目标 revision；
- 同一题目和近似题目有曝光上限；
- 同一知识点连续题数有限制；
- 必须保留探索和课程进度题；
- 连续失败时降低难度、插入先修题或暂停；
- AI 题库补充只通过去标识化 Content Demand；
- 候选 AI 题仍需验证、教师审核和发布门禁。

## 11. 数据收集

收集：

- 版本化 Learning Events；
- 题目—目标映射 revision；
- 状态快照；
- 推荐会话与原因；
- 教师确认、修改和拒绝；
- 前测、后测、延迟测和迁移测；
- 体验、安全和运维指标；
- Feature Flag 与模型版本。

不收集：

- 无关人口属性；
- 社交数据；
- 广告标识符；
- 不必要的完整答案副本；
- 外部 Provider 的学生级画像；
- 未经批准的长期行为追踪。

## 12. 分析计划

实施前冻结：

- 主要和次要指标；
- 排除规则；
- 缺失数据处理；
- 聚类和协变量；
- 多重比较处理；
- 统计模型；
- 效应量和置信区间；
- 分层分析；
- 中止分析；
- 最小有意义差异。

报告同时给出：

- 绝对差异；
- 相对差异；
- 置信区间；
- 样本量；
- 缺失率；
- 实施偏差；
- 负面结果和局限。

不能只报告显著的正向结果。

## 13. 中止与安全规则

满足任一条件时暂停新增推荐并进入人工审查：

- 未经审核题目展示给学生；
- 正式成绩被自动修改；
- 个人画像发送给外部 Generator；
- 跨租户或未授权访问；
- 同意撤回后仍继续处理；
- 低证据误标掌握超过门槛；
- 连续失败或放弃率显著恶化；
- 某群体的难度、机会或学习效果明显退化；
- 数据删除失败；
- 模型无法回放或状态无法解释；
- 监控、审计或 Kill Switch 不可用。

暂停后：

1. 关闭 Feature Flag；
2. 保留底层事件，不继续推荐；
3. 固定受影响版本和事件水位；
4. 通知负责人；
5. 完成影响分析；
6. 修复、回滚或终止试点；
7. 按隐私和事故流程处理数据与通知。

## 14. 教师工作流

试点教师需要：

- 理解状态不是固定能力；
- 查看证据和不确定性；
- 覆盖或冻结错误状态；
- 确认或修改推荐；
- 识别连续失败和学生求助；
- 报告题目、评分和画像问题；
- 不将状态用于处罚或排名；
- 参加定期复盘。

记录教师培训完成情况和支持渠道。

## 15. 学生与监护人体验

说明应使用适龄语言：

- 系统根据近期练习推荐题目；
- 状态会随着新证据变化；
- 不代表永久能力；
- 不影响正式成绩；
- 可以请求教师帮助；
- 可以暂停或退出适用练习；
- 可以询问、更正或申请处理个人数据。

学生端避免展示过度精确概率、负面永久标签和全班排名。

## 16. 技术验收

Stage 1 前：

- Learning Event 幂等和重放通过；
- 教师改分 supersede 通过；
- 同意撤回和删除通过；
- 状态解释 API 可用；
- Feature Flag 和 Kill Switch 可用；
- 跨租户测试通过；
- 模型卡和隐私评估获批。

Stage 3 前：

- `next` 接口幂等；
- 断网恢复不换题；
- 只选择已发布题目；
- 重复曝光和连续失败保护通过；
- 推荐延迟满足目标；
- 监控和告警演练通过；
- 备份恢复后状态可重建；
- 无障碍验收覆盖关键流程。

## 17. 试点退出标准

### 扩大范围

需要同时满足：

- 主要学习指标达到预先声明标准或至少无明显退化；
- 教师明显错误反馈在门槛内；
- 连续失败和退出率无明显恶化；
- 公平性检查无未解决退化；
- 未经审核题目展示率为 0；
- 正式成绩自动修改率为 0；
- 隐私、安全和删除门禁通过；
- 教师和学校同意继续。

### 保持当前范围

如果安全通过但效果不确定，可延长影子模式或教师推荐助手阶段，不自动扩大到学生自适应。

### 终止

若出现无法缓解的隐私、安全、公平性或学习体验风险，则终止试点并执行退出与数据清理计划。
