### 发布原则

1. AI 只创建候选题，不直接发布；
2. 标准答案必须通过独立验证；
3. `blocked` 不得接受，`warning` 必须教师确认；
4. 接受后的草稿仍需运行现有测试和发布门禁；
5. 生成链路不得处理学生个人信息；
6. 模型、Prompt、课程、验证和治理版本必须可追溯；
7. 扩大年级、题型或 Profile 前必须通过总体和分层评估；
8. 默认版本晋级必须绑定正式报告，回滚只能指向历史批准版本；
9. 发布环境验收和学校上线必须分别留存证据。

## 第三阶段：学生知识画像与低风险自适应练习

详细计划：[学生知识画像与自适应练习实施计划](adaptive-learning-plan.md)

### 产品定位

第三阶段根据学生在明确课程目标上的已确认学习证据，估计当前掌握状态、不确定性、遗忘风险和常见错误，并从已经验证、教师审核且发布的题库中选择下一步练习。

系统不提供单一、永久的“个人能力分”，也不把推荐结果直接用于正式成绩、分班、处罚或其他高利害决策。

### 目标链路

```text
作业、练习、复核与订正
→ Learning Event
→ 题目—课程目标证据拆分
→ 学生知识状态与错误模式
→ 教师查看和修正
→ 目标、难度和题目选择
→ 已审核题库中的自适应练习
→ 学习增益与保持度评估
```

题库库存不足时：

```text
库存覆盖缺口
→ 去标识化 Content Demand
→ 第二阶段 AI 候选题生成
→ 独立验证
→ 教师审核
→ 现有发布门禁
→ 进入自适应题池
```

不得把学生身份、原始答案、个人掌握度或知识画像发送给 Generator Provider。

### 进入第三阶段的前置条件

- #37 提供稳定、版本化的 Curriculum Objective Revision；
- #40/#83 能验证进入自适应题池的题目和剩余课程边界；
- #42/#99 建立教师校准、生产事实和版本回归评估；
- #43 建立模型、隐私、配额、审计、回滚和事故治理；
- 题目与课程目标映射经过教师或课程管理员确认；
- 作业、复核、订正和最终评分事件可稳定追溯；
- #29、#30 支持真实英语/数学多题作业；
- 自适应练习明确不计入正式成绩。

### 首期范围

- G4–G9 数学 M1、M2；
- G4–G9 英语 E1、E2；
- E3、E4 仅在教师最终确认后形成知识证据；

### 分阶段发布门禁

#### Gate A：数据与治理基线

- Learning Event 契约和迁移通过；
- 状态可从固定事件水位重建；
- 教师改分可替代旧证据；
- 同意撤回、隐私限制、删除和跨租户测试通过；
- 模型卡、隐私评估和 ADR 获批。

#### Gate B：教师影子画像

- 只对授权教师展示，不向学生展示；
- 教师可查看证据、覆盖和冻结状态；
- 低证据误标、异常跳变和教师纠正率达到门槛；
- Feature Flag 与 Kill Switch 可用。

#### Gate C：教师推荐助手

- 推荐只使用已发布题目；
- 教师确认后才布置；
- 重复、同目标轰炸、难度和连续失败保护通过；
- 推荐原因、策略和模型版本可解释；
- 教师接受、修改和拒绝形成评估数据。

#### Gate D：有限学生自适应试点

- 试点协议批准；
- 学生/监护人说明、退出和人工救济可用；
- 推荐不影响正式成绩；
- 学习效果、体验和公平性没有明显退化；
- 无障碍、监控、告警、备份恢复和事故响应通过；
- 未经审核 AI 题展示率为 0。

### 模型演进

```text
Stage A：可解释规则 + 时间衰减
→ Stage B：BKT/PFA 风格模型
→ Stage C：IRT + 知识追踪
→ Stage D：复杂序列模型实验
```

复杂模型只有在标签、事件量、独立评估、公平性和学习效果均达到门槛后才可进入影子实验。提高下一题预测准确率本身不足以证明模型改善学习。

### 推荐策略边界

首期目标选择包含：当前薄弱目标、先修知识缺口、已掌握但需要复习、当前教学进度，以及探索和跨难度校准。

强制控制：

- 不连续轰炸同一知识点；
- 不因早期错误只提供简单题；
- 连续失败时降低难度、插入先修题或暂停；
- 每次推荐保存原因、证据、策略和模型版本；
- 教师可限制范围、覆盖状态或关闭推荐；
- `next` 接口幂等，网络重试不能改变已分配题目。

### 自适应学习试点门槛

| 指标 | 初始目标 |
| --- | ---: |
| 低证据时错误显示“掌握”的比例 | ≤ 0.5% |
| 单题导致跨越两个以上状态等级 | 0% |
| 教师认为知识状态明显错误 | ≤ 5% |
| 推荐计划教师直接接受率 | ≥ 60% |
| 推荐计划教师修改后接受率 | ≥ 85% |
| 连续失败触发率 | ≤ 10% |
| 近期重复或近似题曝光率 | ≤ 3% |
| 已审核题目使用率 | 100% |
| 未经教师审核 AI 题直接展示率 | 0% |
| 正式成绩被推荐模型自动修改率 | 0% |
| 推荐 API P95 | ≤ 500 ms |

学习增益、延迟保持和迁移效果必须通过对照或准实验评估，不能只看做题量、完成率或下一题预测准确率。

### 发布阶段

1. **事件与映射验证：** 只记录事件和离线计算；
2. **教师影子画像：** 教师查看证据并纠正，不自动推荐；
3. **教师推荐助手：** 教师确认后布置；
4. **低风险自适应练习：** 只用已审核题，不计正式成绩；
5. **库存补充建议：** 提交去标识化题库需求；
6. **扩大范围：** 只有学习效果、公平性和教师纠正率稳定后扩展。

## 总体发布底线

1. 未经教师审核的 AI 题不能展示给学生；
2. `needs_review` 或依赖异常不能作为强掌握证据；
3. 学生个人画像不能发送给外部生成模型；
4. 自动推荐不能修改正式成绩；
5. 教师和学生可以查看主要判断依据并获得人工救济；
6. 所有课程、题目、模型、Prompt、状态和推荐版本必须可追溯；
7. 扩大自动化前必须证明没有明显学习效果、公平性或安全退化；
8. 仓库 CI、发布环境验收和生产上线必须分别保留证据。
