# 产品路线图

本路线图区分四个层次：已经建设的作业批改 MVP、当前试点准备任务、K–13 课程约束型 AI 出题，以及后续的学生知识画像与低风险自适应练习。

详细设计与验收标准见：

- [K–13 AI 出题实施计划](ai-question-generation-plan.md)
- [学生知识画像与自适应练习实施计划](adaptive-learning-plan.md)

## 第一阶段：英语与数学作业批改 MVP

| 周期 | 目标 | 主要交付 |
| --- | --- | --- |
| 1–2 | 需求冻结与技术验证 | 题型、原型、安全 AST、判分样本 |
| 3–4 | 平台基础 | 用户、班级、租户、题库、作业、审计 |

首条垂直链路是：教师创建数学题 → 学生输入 → 系统判分 → 教师查看证据并发布 → 学生申诉/订正。

### 发布原则

1. AI 只创建候选题，不直接发布；
2. 标准答案必须通过独立验证；
3. `blocked` 题不得接受，`warning` 必须教师确认；
4. 接受后的题目仍需运行现有测试用例和发布门禁；
5. 生成链路不得处理学生个人信息；
6. 模型、Prompt、课程和验证版本必须可追溯；
7. 扩大年级、题型或课程 profile 前必须通过分层评估。

### AI 出题质量门槛

| 指标 | 初始目标 |
| --- | ---: |
| 结构 Schema 通过率 | ≥ 98% |
| 数学答案错误率 | ≤ 0.5% |
| 年级明显不匹配率 | ≤ 2% |
| 重复或高度相似题率 | ≤ 3% |
| 教师直接接受率 | ≥ 60% |
| 教师修改后接受率 | ≥ 85% |
| 发布后改答案率 | ≤ 0.5% |
| 未经教师审核发布率 | 0% |

这些阈值必须由版本化评估集和影子试点校准，不作为不可变常量硬编码。

## 第三阶段：学生知识画像与低风险自适应练习

详细计划：[学生知识画像与自适应练习实施计划](adaptive-learning-plan.md)

### 产品定位

第三阶段根据学生在明确课程目标上的已确认学习证据，估计当前掌握状态、不确定性、遗忘风险和常见错误，并从已经验证、教师审核且发布的题库中选择下一步练习。

系统不提供单一、永久的“个人能力分”，也不把推荐结果直接用于正式成绩、分班、处罚或高利害决策。

### 目标链路

```text
作业、练习、复核与订正
→ Learning Event
→ 题目—课程目标证据拆分
→ 学生知识状态与错误模式
→ 教师查看和修正
→ 目标、难度和题目选择
→ 已审核题库中的自适应练习
→ 学习增益与保持度评估
```

题库库存不足时：

```text
库存覆盖缺口
→ 去标识化 content demand
→ 第二阶段 AI 候选题生成
→ 独立验证
→ 教师审核
→ 现有发布门禁
→ 进入自适应题池
```

不得把学生身份、原始答案、个人掌握度或知识画像发送给 Generator provider。

### 进入第三阶段的前置条件

- #37 提供稳定、版本化的 curriculum objective revision；
- #40 能验证进入自适应题池的题目；
- #42 建立分层质量与版本回归评估；
- #43 建立模型、隐私、配额和审计治理；
- 题目与课程目标映射经过教师或课程管理员确认；
- 作业、复核、订正和最终评分事件可稳定追溯；
- #29、#30 支持真实英语/数学多题作业；
- 自适应练习明确不计入正式成绩。

### 首期范围

- G4–G9 数学 M1、M2；
- G4–G9 英语 E1、E2；
- E3、E4 仅在教师最终确认后形成知识证据；
- 先交付教师影子画像和推荐助手，再开放学生自适应练习；
- K 和 G13 不进入首个自适应试点；
- 自适应系统只使用已发布题目。

### 16 周计划

| 周期 | 目标 | 主要交付 |
| --- | --- | --- |
| 1–2 | 证据基础 | 题目—目标映射、Learning Event、最终证据规则、迁移和审计 |
| 3–4 | 状态模型 | 可解释掌握度、置信度、状态快照、回放工具和影子计算 |
| 5–6 | 教师洞察 | 班级热力图、学生详情、证据解释和人工覆盖 |
| 7–8 | 推荐助手 | 目标选择、难度带、题目过滤和教师确认式练习计划 |
| 9–11 | 低风险自适应 | 学生练习会话、幂等选题、连续失败保护、暂停和恢复 |
| 10–12 | 库存联动 | 覆盖矩阵、去标识化补题需求和 AI 出题工作台联动 |
| 12–14 | 评估与治理 | 离线回放、学习增益、延迟保持、公平性、删除和人工救济 |
| 15 | 教师影子试点 | 推荐只展示给教师，不自动派题，收集纠正和修改原因 |
| 16 | 有限学生试点 | 仅开放通过门槛的年级、目标和题型，不影响正式成绩 |

### 模型演进

```text
Stage A：可解释规则 + 时间衰减
→ Stage B：BKT/PFA 风格模型
→ Stage C：IRT + 知识追踪
→ Stage D：复杂序列模型实验
```

复杂模型只有在标签、事件量、独立评估、公平性和学习效果均达到门槛后才可进入影子实验。提高下一题预测准确率本身不足以证明模型改善学习。

### 推荐策略边界

首期目标选择包含：

- 当前薄弱目标；
- 先修知识缺口；
- 已掌握但需要复习；
- 当前教学进度；
- 探索和跨难度校准。

强制控制：

- 不连续轰炸同一知识点；
- 不因早期错误只提供简单题；
- 连续失败时降低难度、插入先修题或暂停；
- 每次推荐保存原因、证据、策略和模型版本；
- 教师可限制范围、覆盖状态或关闭推荐；
- `next` 接口幂等，网络重试不能改变已分配题目。

### 自适应学习试点门槛

以下是需要通过影子数据校准的初始目标：

| 指标 | 初始目标 |
| --- | ---: |
| 无证据或低证据时错误显示“掌握”的比例 | ≤ 0.5% |
| 单题导致跨越两个以上状态等级的比例 | 0% |
| 教师认为知识状态明显错误的比例 | ≤ 5% |
| 推荐计划教师直接接受率 | ≥ 60% |
| 推荐计划教师修改后接受率 | ≥ 85% |
| 自适应会话连续失败触发率 | ≤ 10% |
| 近期重复或近似题曝光率 | ≤ 3% |
| 已审核题目使用率 | 100% |
| 未经教师审核 AI 题直接展示率 | 0% |
| 正式成绩被推荐模型自动修改率 | 0% |
| 推荐 API P95 | ≤ 500 ms |

学习增益、延迟保持和迁移效果必须通过对照或准实验评估，不能只看做题量、完成率或下一题预测准确率。

### 发布阶段

1. **事件与映射验证：** 只记录事件和离线计算，不展示画像；
2. **教师影子画像：** 教师查看证据并纠正，不自动推荐；
3. **教师推荐助手：** 教师确认后布置；
4. **低风险自适应练习：** 仅课后练习，只用已审核题，不计正式成绩；
5. **库存补充建议：** 向 AI 出题系统提交去标识化题库需求；
6. **扩大范围：** 只有学习效果、公平性和教师纠正率稳定后才扩展。

## 总体发布底线

无论处于哪个阶段，以下底线保持不变：

1. 未经教师审核的 AI 题不能展示给学生；
2. `needs_review` 或依赖异常结果不能作为强掌握证据；
3. 学生个人画像不能发送给外部生成模型；
4. 自动推荐不能修改正式成绩；
5. 教师和学生可以查看主要判断依据并获得人工救济；
6. 所有课程、题目、模型、Prompt、状态和推荐版本必须可追溯；
7. 扩大自动化程度前必须证明没有明显学习效果、公平性或安全退化。
