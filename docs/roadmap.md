# 16 周路线图

| 周期 | 目标 | 主要交付 |
|---|---|---|
| 1–2 | 需求冻结与技术验证 | 题型、原型、安全 AST、判分样本 |
| 3–4 | 平台基础 | 用户、班级、租户、题库、作业、审计 |
| 5–6 | 作答闭环 | 学生作答、自动保存、提交、教师发布 |
| 7–8 | 数学批改 | 数值、表达式、方程、步骤、反馈 |
| 9–10 | 英语批改 | 规范化、词形、语法、评分点、复核 |
| 11 | 复核与订正 | 队列、改分理由、申诉、错题报告 |
| 12 | 验收准备 | 性能、安全、备份、培训与手册 |
| 13–16 | 试点 | 影子模式、有限自动发布、指标复盘 |

首个里程碑应完成一条真实链路：教师创建数学题 → 学生手机输入 → 系统判分 → 教师查看证据 → 学生订正。
