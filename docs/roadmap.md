### 发布原则

1. AI 只创建候选题，不直接发布；
2. 标准答案必须通过独立验证；
3. `blocked` 不得接受，`warning` 必须教师确认；
4. 接受后的草稿仍需运行现有测试和发布门禁；
5. 生成链路不得处理学生个人信息；
6. 模型、Prompt、课程、验证和治理版本必须可追溯；
7. 扩大年级、题型或 Profile 前必须通过总体和分层评估；
8. 默认版本晋级必须绑定正式报告，回滚只能指向历史批准版本；
9. 发布环境验收和学校上线必须分别留存证据。

### AI 出题质量门槛

| 指标 | 初始目标 |
| --- | ---: |
| Schema 通过率 | ≥ 98% |
| 数学答案错误率 | ≤ 0.5% |
| 年级明显不匹配率 | ≤ 2% |
| 重复或高度相似题率 | ≤ 3% |
| 教师直接接受率 | ≥ 60% |
| 教师修改后接受率 | ≥ 85% |
| 发布后改答案率 | ≤ 0.5% |
| 未经教师审核发布率 | 0% |

这些阈值是待教师黄金集和影子试点校准的起始目标，不作为不可修改常量。

## 第三阶段：学生知识画像与低风险自适应练习

详细计划：[学生知识画像与自适应练习实施计划](adaptive-learning-plan.md)

### 产品定位

第三阶段根据学生在明确课程目标上的已确认学习证据，估计当前掌握状态、不确定性、遗忘风险和常见错误，并从已经验证、教师审核且发布的题库中选择下一步练习。

系统不提供单一、永久的“个人能力分”，也不把推荐结果直接用于正式成绩、分班、处罚或其他高利害决策。

### 目标链路

```text
作业、练习、复核与订正
→ Learning Event
→ 题目—课程目标证据拆分
→ 学生知识状态与错误模式
→ 教师查看和修正
→ 目标、难度和题目选择
→ 已审核题库中的自适应练习
→ 学习增益与保持度评估
```

题库库存不足时：

```text
库存覆盖缺口
→ 去标识化 Content Demand
→ 第二阶段 AI 候选题生成
→ 独立验证
→ 教师审核
→ 现有发布门禁
→ 进入自适应题池
```

不得把学生身份、原始答案、个人掌握度或知识画像发送给 Generator Provider。

### 进入第三阶段的前置条件

- #37 提供稳定、版本化的 Curriculum Objective Revision；
- #40/#83 能验证进入自适应题池的题目和剩余课程边界；
- #42 建立按年级、学科、题型、难度和 Profile 分层的质量门禁；
- #43 提供模型、Prompt、预算、隐私和版权治理；
- #31/#33 完成真实身份、数据库、Grader、备份恢复和发布环境验收；
- 教学负责人、隐私负责人和试点运维负责人批准进入影子模式。

### 阶段门禁

#### Gate A：数据与治理基线

- Learning Event、题目—目标映射和学生状态契约已评审；
- 教师最终确认结果优先于自动评分证据；
- 自适应练习不修改正式成绩；
- 学生画像不发送给外部 Generator Provider；
- 隐私影响评估、保留、删除和申诉机制已批准。

#### Gate B：教师影子画像

- 只向授权教师显示；
- 状态包含证据数量、置信度和不确定性；
- 教师可以解释、覆盖、冻结和撤销；
- 状态可以从不可变 Learning Events 重建；
- 学生端尚不显示自动推荐。

#### Gate C：教师推荐助手

- 推荐只选择已验证、已审核、已发布题目；
- 教师可以预览、修改和确认；
- 连续失败保护、近期去重和探索配额生效；
- 推荐理由和策略版本可回放；
- 不进入正式成绩。

#### Gate D：有限学生自适应试点

- 主要学习增益和保持度指标已定义；
- 连续失败、跳过、退出和暂停规则已通过演练；
- 公平性和学习机会差异已检查；
- 教师、学生与监护人说明已批准；
- 技术、安全或体验中止条件已固化。

### 建议实施工作流

- P0 Learning Data：事件、证据与映射；
- P0 Student Model：掌握状态、不确定性和重建；
- P1 Diagnosis & Calibration：错误模式和题目质量；
- P1 Recommendation：目标、难度、题型和题目选择；
- P1 Teacher Web：洞察、解释和人工控制；
- P1 Student Web：知识地图和练习体验；
- P1 Inventory：题库覆盖和去标识化补题；
- P1 Evaluation & Governance：学习效果、公平性和治理。

### 明确禁止

- 用学生知识画像自动分班、升留级、处罚或招生；
- 把推荐练习结果直接写入正式成绩；
- 用单道题形成永久能力标签；
- 把学生身份、班级、成绩、作答或画像发送给外部模型；
- AI 生成题目绕过验证、教师审核和原有发布门禁；
- 在 E3、E4 等主观题上取消教师最终确认；
- 以参与度、接受率或模型置信度替代真实学习效果。
