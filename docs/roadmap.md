# 产品路线图

本路线图区分已经建设的作业批改 MVP、当前试点准备任务和下一阶段 K–13 课程约束型 AI 出题。详细设计与验收标准见 [K–13 AI 出题实施计划](ai-question-generation-plan.md)。

## 第一阶段：英语与数学作业批改 MVP

| 周期 | 目标 | 主要交付 |
| --- | --- | --- |
| 1–2 | 需求冻结与技术验证 | 题型、原型、安全 AST、判分样本 |
| 3–4 | 平台基础 | 用户、班级、租户、题库、作业、审计 |
| 5–6 | 作答闭环 | 学生作答、自动保存、提交、教师发布 |
| 7–8 | 数学批改 | 数值、表达式、方程、步骤、反馈 |
| 9–10 | 英语批改 | 规范化、词形、语法、评分点、复核 |
| 11 | 复核与订正 | 队列、改分理由、申诉、错题报告 |
| 12 | 验收准备 | 性能、安全、备份、培训与手册 |
| 13–16 | 试点 | 影子模式、有限自动发布、指标复盘 |

首条垂直链路是：教师创建数学题 → 学生输入 → 系统判分 → 教师查看证据并发布 → 学生申诉/订正。

### 发布原则

1. AI 只创建候选题，不直接发布；
2. 标准答案必须通过独立验证；
3. `blocked` 题不得接受，`warning` 必须教师确认；
4. 接受后的题目仍需运行现有测试用例和发布门禁；
5. 生成链路不得处理学生个人信息；
6. 模型、Prompt、课程和验证版本必须可追溯；
7. 扩大年级、题型或课程 profile 前必须通过分层评估。

## 建议质量门槛

| 指标 | 初始目标 |
| --- | ---: |
| 结构 Schema 通过率 | ≥ 98% |
| 数学答案错误率 | ≤ 0.5% |
| 年级明显不匹配率 | ≤ 2% |
| 重复或高度相似题率 | ≤ 3% |
| 教师直接接受率 | ≥ 60% |
| 教师修改后接受率 | ≥ 85% |
| 发布后改答案率 | ≤ 0.5% |
| 未经教师审核发布率 | 0% |

这些阈值必须由版本化评估集和影子试点校准，不作为不可变常量硬编码。
