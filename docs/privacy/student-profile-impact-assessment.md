# 学生知识画像隐私影响评估

状态：Draft for review  
适用范围：第三阶段学生知识画像、教师推荐助手和低风险自适应练习  
不构成法律意见；正式试点前需要由产品、学校、隐私/合规和安全负责人共同审批。

## 1. 评估对象

本评估覆盖以下处理活动：

- 将作业、练习、复核和订正转换为 Learning Events；
- 将题目证据映射到课程目标 revision；
- 计算学生在各课程目标上的知识状态、不确定性和遗忘风险；
- 识别经规则或教师确认的常见错误模式；
- 向教师展示个人和班级洞察；
- 生成教师确认式练习建议；
- 在有限试点中向学生提供低风险自适应练习；
- 统计题库覆盖缺口并向 AI 出题系统发送去标识化内容需求；
- 运行模型评估、公平性检查和受控实验。

## 2. 处理目的

允许的目的：

1. 帮助教师理解学生在明确课程目标上的当前学习证据；
2. 识别证据不足、先修缺口、待巩固和建议复习目标；
3. 选择已审核题目形成低风险课后练习；
4. 评估学习增益、延迟保持和教学覆盖；
5. 提升题库覆盖和教师备课效率；
6. 发现模型、题目或评分规则的系统性错误。

禁止的目的：

- 推断智力、人格、心理健康、家庭能力或社会价值；
- 自动修改正式成绩；
- 自动分班、升留级、招生、处罚或淘汰；
- 形成公开排名；
- 向广告、营销或无关第三方提供画像；
- 用个人画像训练或提示外部 AI Generator；
- 对教师进行个体绩效监控；
- 在没有新的审批和告知时扩展到高利害决策。

## 3. 数据清单

### 3.1 原始业务数据

| 数据 | 用途 | 是否进入学生模型 |
| --- | --- | --- |
| 题目版本与评分策略 | 解释证据 | 是，按引用 |
| 学生答案 | 原始评分和复核 | 不直接进入模型；只保留引用或结构化摘要 |
| GradingRun | 自动评分证据 | 条件进入 |
| ReviewDecision | 教师最终证据 | 是，优先级最高 |
| GradePublication | 正式发布事实 | 是 |
| CorrectionAttempt | 订正证据 | 是 |
| 提示与尝试次数 | 证据强度 | 是，最小化 |
| 响应时间 | 辅助信号 | 仅在可靠采集且经过审批时 |

### 3.2 派生数据

- Learning Events；
- objective evidence；
- student objective states；
- 状态快照；
- misconception states；
- recommendation sessions；
- experiment assignments；
- content demand；
- 聚合的班级和版本评估指标。

### 3.3 明确排除的数据

- 姓名之外的无关身份信息；
- 手机号、邮箱和家庭地址；
- 家庭收入、父母职业、民族、健康和生物识别数据；
- 社交媒体数据；
- 教师私人评价；
- 广告标识符；
- 与学习目的无关的设备指纹；
- 外部来源购买的个人画像。

## 4. 数据流

```text
学生作答
→ Core API / Grader / 教师复核
→ 正式业务记录
→ 事务 Outbox
→ Learning Event
→ Objective Evidence
→ Student Model
→ 状态快照
→ 教师端解释与推荐助手
→ 可选学生端低风险练习
```

题库不足时：

```text
聚合覆盖缺口
→ 去标识化 Content Demand
→ AI Generator
→ 候选题验证
→ 教师审核
→ 正式题库
```

Content Demand 不得包含学生 ID、班级 ID、原始答案、掌握概率、画像状态或成绩。它只能在获批的
聚合策略版本下生成，且必须覆盖至少 10 名不同学生和 3 个班级；低于该阈值时必须抑制请求或粗化
课程目标、难度和错误模式，不能将稀有学习缺口发送到外部 Generator。

## 5. 角色与访问控制

### 学生

可以：

- 查看自己的解释性知识状态；
- 查看主要证据和推荐理由；
- 请求教师复核或更正；
- 暂停或退出非强制作业的自适应练习；
- 访问适用的数据查阅、更正和删除流程。

不能：

- 查看其他学生；
- 查看班级个人排名；
- 查看内部模型参数、安全规则或他人事件。

### 教师

可以：

- 查看自己负责班级内学生；
- 查看证据、状态历史和错误模式；
- 覆盖、冻结或恢复某个状态；
- 确认推荐计划；
- 查看班级聚合热力图。

不能：

- 查看未授权班级；
- 将画像用于未批准的高利害决策；
- 绕过审计批量导出完整个人画像。

### 管理员与运维

遵循最小权限。数据库、日志和备份访问必须审计。支持人员默认只访问脱敏诊断信息。

### 模型与外部 Provider

外部 Generator 只能接收去标识化课程和内容需求。个人 Student Model 状态不得离开平台受控边界。

## 6. 主要风险与缓解措施

### 6.1 将短期表现误认为固定能力

风险：学生因少量错误被长期标记为薄弱。

缓解：

- 低证据显示 `insufficient_evidence`；
- 单题状态变化上限；
- 保留跨难度探索；
- 状态随新证据更新；
- 教师覆盖和学生人工救济；
- 页面避免“能力不足”等永久标签。

### 6.2 自动评分错误污染画像

风险：错误批改导致错误推荐。

缓解：

- 教师最终确认优先；
- E3/E4 未复核前不形成强证据；
- `needs_review`、依赖异常和 Grader 故障排除；
- 支持事件 supersede 和全量重放；
- 监控教师改分率和画像覆盖率。

### 6.3 反馈闭环降低学习机会

风险：系统认为学生能力低，只提供简单题，使学生无法证明进步。

缓解：

- 固定探索和校准比例；
- 课程进度题；
- 难度范围上下限；
- 比较群体间难度和机会分布；
- 教师可以限制或关闭推荐；
- 连续失败保护不等于永久降级。

### 6.4 未成年人画像泄露

风险：知识状态、错误模式或成绩被未授权访问。

缓解：

- 租户、班级和角色隔离；
- HttpOnly 会话与 CSRF 防护；
- 个人导出额外权限；
- 日志不记录完整画像；
- 加密传输和受控备份；
- 访问审计和异常告警；
- 定期权限复核。

### 6.5 目的扩张

风险：画像被用于分班、处罚或营销。

缓解：

- 模型卡和 ADR 明确禁止用途；
- API 不提供高利害自动决策端点；
- Feature Flag 按租户控制；
- 使用场景变更需要重新评估；
- 审计导出和管理员操作。

### 6.6 外部模型获得个人数据

风险：个人画像或答案进入第三方 Provider。

缓解：

- Processor Policy 禁止身份和画像字段；
- Generator 仅接收 Content Demand；
- Provider 主机 allowlist；
- 出站载荷测试；
- 普通日志不记录模型完整请求；
- 配置错误 fail closed。

### 6.7 公平性退化

风险：不同群体收到系统性不同的难度、探索或课程覆盖。

缓解：

- 在线模型不使用人口属性；
- 受控离线公平性审计；
- 按年级、课程 profile、题型、设备和证据量分层；
- 检查难度、探索机会、提示和连续失败；
- 发现退化时暂停推荐并回滚版本。

### 6.8 删除不完整

风险：原账号删除后，事件、状态或实验记录仍保留可识别关系。

缓解：

- 建立派生数据清单和删除图；
- 删除流程覆盖事件、状态、快照、错误模式、推荐和实验；
- 备份恢复后重放删除清单；
- 自动化完整性检查；
- 法律冻结与删除请求状态分离。

## 7. 数据最小化

设计要求：

- Student Model 使用内部 UUID，不使用姓名；
- Learning Event 保存引用和稳定码，不复制答案全文；
- 响应时间默认不进入模型；只有在可靠采集、用途审批已版本化且模型卡明确启用时才可采集和使用；
- 教师自由文本备注默认不进入模型；
- 班级报告默认聚合；
- 实验分析优先使用去标识化数据；
- 日志、Tracing 和错误上报不包含答案、画像或 Token；
- Content Demand 不包含个人数据。

每个新增字段需要记录：用途、访问角色、保留期限、是否必要以及删除方式。

## 8. 准确性、更正和人工救济

平台需要提供：

- 每个状态的主要证据；
- 证据时间和题型；
- 不确定性提示；
- 教师覆盖入口；
- 学生请求复核入口；
- 覆盖原因和审计；
- 撤销覆盖和状态重建；
- 发现系统错误时批量重算受影响学生的能力。

不得仅显示一个不可解释的百分比。

## 9. 同意、告知和退出

试点前需要为学校、教师、学生和监护人提供适龄说明：

- 为什么创建知识状态；
- 使用哪些学习证据；
- 不用于哪些决策；
- 谁能查看；
- 如何更正；
- 如何退出低风险自适应练习；
- 数据保存多久；
- 是否使用外部处理器；
- 如何提出隐私请求。

同意撤回或处理限制生效后：

- 停止新的状态更新；
- 停止推荐；
- 阻止对外处理；
- 按政策处理派生数据；
- 不影响依法需要保留的正式成绩记录；
- 提供明确状态和人工处理路径。

## 10. 保留与删除

保留期限需要由学校和业务场景确认。默认原则：

- 原始正式成绩按学校政策保留；
- Learning Events 不应无限期保留；
- 高频状态快照只保留必要版本；
- 推荐会话和实验记录短于正式成绩；
- 聚合、不可逆去标识化指标可以延长保留；
- 退出试点后执行明确的数据清理计划。

删除应支持：

- 单学生；
- 单租户；
- 指定时间窗口；
- 指定实验；
- Provider 或模型事故影响范围。

## 11. 安全控制

- 最小权限和定期访问复核；
- 数据库和备份加密；
- HTTPS；
- Secret Manager；
- 生产默认密钥拒绝；
- 出站处理器 allowlist；
- 审计链；
- 异常下载和批量访问告警；
- Feature Flag 和 Kill Switch；
- 备份恢复与删除重放演练；
- 事故响应和通知流程。

## 12. 模型与版本治理

每次状态与推荐保存：

- 模型 ID/version；
- 配置版本；
- 课程目标 revision；
- 题目映射 revision；
- 证据游标；
- 推荐策略版本；
- Feature Flag 状态。

模型升级需要：

- 离线重放；
- 差异报告；
- 公平性检查；
- 教师抽样审核；
- 影子运行；
- 灰度和回滚计划。

## 13. 外部处理器

首期 Student Model 和推荐服务应在平台受控环境运行。

任何外部处理器接入前需要记录：

- 服务名称和区域；
- 数据类别；
- 数据保留和训练使用条款；
- 安全认证；
- 子处理器；
- 删除机制；
- 合同和学校批准；
- 退出和迁移计划。

未经审批不得将个人状态、事件或答案发送给外部服务。

## 14. 试点门禁

教师影子模式前：

- Learning Event 契约评审通过；
- 模型卡评审通过；
- 跨租户、同意撤回和删除测试通过；
- 教师可以查看证据并更正；
- 普通日志脱敏检查通过；
- Kill Switch 可用；
- 本评估由隐私/合规和安全负责人签署。

有限学生试点前：

- 学生/监护人说明可用；
- 退出和人工救济可用；
- 推荐不影响正式成绩；
- 只使用已发布题目；
- 公平性和连续失败保护通过；
- 试点实验协议批准；
- 事故响应和值班联系人明确。

## 15. 待决策项

正式批准前需要确认：

1. 试点地区、学校和适用规则；
2. 各类派生数据保留期限；
3. 学生和监护人告知/同意形式；
4. 教师导出个人画像的权限边界；
5. 响应时间是否进入模型；
6. 公平性审计所需受保护属性及隔离方式；
7. 退出自适应练习后是否保留教师只读历史；
8. 试点中止和数据清理责任人。
