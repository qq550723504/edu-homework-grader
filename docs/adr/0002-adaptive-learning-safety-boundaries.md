# ADR-0002：自适应学习的证据、推荐与安全边界

状态：Proposed  
日期：2026-07-21

## 背景

平台已经具备作业、题库、自动批改、教师复核、成绩发布和订正能力，并计划建设 K–13 AI 出题与学生知识画像。

自适应学习会把学生表现转化为知识状态并选择下一步练习。如果没有明确架构边界，后续实现可能出现：

- 从可变聚合表反推状态，无法重放或解释；
- 把未复核自动评分当成最终教学事实；
- 点击“下一题”时实时生成未经审核题目；
- 将学生画像发送给外部 Generator；
- 让推荐结果影响正式成绩或高利害决策；
- 因早期错误只提供低难度题，形成自我强化闭环；
- 模型升级后历史状态无法复现或回滚。

因此需要将不可轻易改变的边界固化为架构决策。

## 决策

### 1. 学生知识状态由不可变 Learning Events 派生

原始业务记录产生版本化、幂等、可替代的 Learning Events。学生知识状态和推荐均为派生数据，可从固定事件水位和指定模型版本重建。

不允许只保存当前掌握度而没有可回放证据。

### 2. 教师最终确认优先于自动评分

证据优先级为：

1. 教师最终确认；
2. 已发布订正；
3. 已发布正式结果；
4. 高置信度确定性自动判分；
5. provisional 系统建议。

`needs_review`、依赖异常、未发布主观题和 Grader 故障不形成强掌握证据。

### 3. 题目—课程目标映射必须审核并版本化

推荐和状态计算只能使用审核过的 mapping revision。修改映射不改变历史事件的解释，后续事件引用新 revision。

### 4. 推荐只选择已验证、教师审核并发布的题目

学生请求下一题时不得调用 Generator 生成并直接展示新题。

题库不足时只生成去标识化 Content Demand，由 AI 出题流程生成候选题，经过独立验证、教师审核和现有发布门禁后进入题池。

### 5. 个人画像不得发送给外部 Generator Provider

Generator 只能接收课程目标、题型、难度、错误模式和库存数量等去标识化内容需求。

禁止发送：

- 学生 ID；
- 班级 ID；
- 原始答案；
- 成绩；
- 掌握概率；
- 状态标签；
- 推荐历史；
- Access/Refresh Token。

### 6. 自适应练习与正式成绩分离

首期自适应能力只用于低风险课后练习、订正和教师推荐助手。模型和推荐 API 没有修改正式成绩的权限。

任何高利害用途需要新的 ADR、模型卡、隐私影响评估和审批。

### 7. 保留探索机会和连续失败保护

推荐策略必须：

- 保留探索和跨难度校准；
- 不连续轰炸同一知识点；
- 限制近期重复；
- 连续失败时降难度、插入先修题、提示或暂停；
- 不因早期错误永久降低难度；
- 保存每次推荐原因和策略版本。

### 8. 教师拥有解释、覆盖和暂停权

教师可以查看主要证据、覆盖或冻结状态、确认或修改练习计划，并关闭某学生/班级推荐。

覆盖不删除底层事件，必须审计且可撤销。

### 9. 模型、配置和推荐版本可回滚

每个状态快照和推荐会话保存：

- 模型 ID/version；
- 配置版本；
- 课程目标 revision；
- 映射 revision；
- 证据游标；
- 策略版本；
- Feature Flag 状态。

升级先离线重放、影子运行和灰度。回滚切换读取版本，不改写事件。

## 后果

### 正面后果

- 状态可解释、可审计、可重放；
- 教师改分能够纠正画像；
- AI 出题与个人画像保持隐私隔离；
- 未经审核题目不会进入学生练习；
- 模型升级和错误修复可回滚；
- 降低高利害误用风险；
- 可以评估学习增益而非只看预测准确率。

### 成本与复杂度

- 需要事务 Outbox、事件去重和重放工具；
- 需要版本化 mapping、状态快照和模型配置；
- 教师端需要证据与覆盖界面；
- 推荐服务需要题库过滤、曝光控制和幂等会话；
- 删除流程需要覆盖更多派生数据；
- 真实试点前需要隐私、评估、公平性和运维门禁。

这些成本被认为是处理未成年人教育画像所必需的。

## 被否决的方案

### 方案 A：只保存一个学生总能力分

否决原因：无法解释具体知识点、证据不足和错误模式，容易形成固定标签，也无法可靠选题。

### 方案 B：直接使用未复核自动评分更新画像

否决原因：主观题、依赖故障和规则错误会污染状态，教师改分后难以纠正。

### 方案 C：实时调用 AI 为个人生成下一题

否决原因：无法保证答案正确、年龄适配、版权和安全，也会把个人画像暴露给 Provider，并绕过教师审核。

### 方案 D：让复杂深度模型直接进入生产

否决原因：数据量、映射质量、独立评估和公平性尚未满足。首期选择可解释模型，复杂模型只进入影子实验。

### 方案 E：推荐直接计入正式成绩

否决原因：练习难度与路径因人而异，不适合作为未经额外设计的可比正式评价。

## 合规与验证

本 ADR 在以下条件满足后可改为 Accepted：

- Learning Event v1 契约评审通过；
- Student Model v1 模型卡评审通过；
- 学生画像隐私影响评估获批；
- 教师覆盖、Feature Flag 和 Kill Switch 原型验证；
- 跨租户、同意撤回、删除和出站载荷测试通过；
- 试点实验协议获批。

## 变更规则

修改以下任何决策需要新的 ADR，而不是直接编辑本文件：

- 允许外部 Provider 获取个人画像；
- 允许未经审核 AI 题实时展示；
- 允许自适应结果修改正式成绩；
- 改变教师最终证据优先级；
- 不再从不可变事件重建状态；
- 将系统用于分班、招生、处罚或其他高利害决策。
