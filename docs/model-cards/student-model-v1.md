# Student Model v1 模型卡

状态：Proposed  
模型族：可解释规则与时间衰减  
适用范围：G4–G9 数学 M1/M2、英语 E1/E2 的影子画像与低风险练习

## 1. 模型目的

Student Model v1 根据版本化 Learning Events 和题目—课程目标映射，估计学生在每个课程目标上的当前知识状态、证据充分度、遗忘风险和常见错误。

模型输出用于：

- 教师查看班级和个人知识点证据；
- 识别证据不足、待巩固和建议复习的目标；
- 为教师生成可解释的练习建议；
- 在教师确认和试点门禁通过后，为低风险课后练习选择已审核题目；
- 评估课程目标覆盖和题库库存缺口。

模型不输出人格、智力或永久能力结论。

## 2. 明确禁止用途

Student Model v1 不得用于：

- 自动修改正式成绩；
- 分班、升留级、招生、处罚或淘汰；
- 教师绩效评价；
- 生成公开或班级排名；
- 推断家庭背景、健康、性格或其他非学习目标属性；
- 将个人知识画像发送给外部 AI Generator；
- 在缺少教师和合规审批时自动给学生布置高强度练习；
- 作为心理诊断或认知能力测试。

任何超出本模型卡的用途都需要新的隐私影响评估、产品审批和版本化模型卡。

## 3. 输入数据

模型只读取：

- `learning-event-v1`；
- 已审核的题目—课程目标映射 revision；
- 题目类型、策略版本、难度与校准元数据；
- 教师最终确认的评分、错误原因和订正结果；
- 提示次数、尝试次数和可靠的响应时间摘要；
- 课程目标先修关系；
- 模型配置和时间衰减参数。

模型不读取：

- 姓名、邮箱、手机号或学校编号；
- 性别、民族、家庭收入、地区或父母职业；
- 外部社交数据；
- 未经允许的浏览器行为轨迹；
- 学生自由文本答案全文，除非某个受控诊断器在业务层先转换为稳定错误码；
- 教师自由文本备注全文；
- 外部 AI 的未验证主观判断。

## 4. 输出

### 4.1 数值输出

| 字段 | 范围 | 含义 |
| --- | --- | --- |
| `mastery_probability` | 0–1 | 当前模型下的掌握估计，不是客观事实 |
| `confidence` | 0–1 | 证据充分度和一致性 |
| `forgetting_risk` | 0–1 | 基于时间和历史稳定性的复习风险 |
| `evidence_count` | integer | 当前有效证据数 |
| `effective_evidence_weight` | decimal | 经过可信度、难度和时间衰减后的证据总量 |

### 4.2 展示状态

面向教师和学生优先展示以下状态，而不是精确概率：

- `insufficient_evidence`：证据不足；
- `needs_support`：需要支持；
- `developing`：正在发展；
- `mastered`：已掌握；
- `stable_mastery`：稳定掌握；
- `review_due`：建议复习；
- `teacher_overridden`：教师已覆盖。

阈值属于模型配置，必须版本化和可回放，不得散落在前端。

## 5. v1 算法概述

v1 使用可解释的证据聚合与时间衰减，不使用端到端深度序列模型。

计算步骤：

```text
有效 Learning Events
→ 按课程目标拆分证据
→ 应用事件可信度
→ 应用题目难度与映射权重
→ 应用提示/重试折减
→ 应用时间衰减
→ 计算正负证据和冲突度
→ 更新 mastery_probability 与 confidence
→ 映射为展示状态
```

### 5.1 证据方向

- 独立完成、教师确认正确：强正证据；
- 高难度确定性题正确：较强正证据；
- 使用提示后正确：弱正证据；
- 多次重试后正确：弱正证据；
- 明确目标步骤错误：强负证据；
- 非目标计算错误：弱负证据或不计入；
- 未复核主观题：不计入或 provisional；
- `grader-unavailable`：不计入。

### 5.2 时间衰减

衰减参数按 curriculum profile、学科、目标类型配置。首期不假设所有知识点具有相同遗忘速度。

衰减只影响当前状态，不改写历史事件。模型升级或参数变化通过重放生成新快照。

### 5.3 单事件变化限制

v1 必须限制单个事件对状态的最大影响：

- 一道题不能让状态跨越两个以上展示等级；
- 首次正确不能直接进入 `stable_mastery`；
- 首次错误不能直接形成永久 `needs_support`；
- 教师明确覆盖除外，但需要原因和审计。

## 6. 冷启动

新学生或新目标没有足够证据时：

- 状态显示 `insufficient_evidence`；
- 不使用默认 0.5 伪装成已估计掌握度；
- 参考年级、课程进度和先修目标仅作为选题先验，不作为掌握事实；
- 首批练习包含低压力诊断、先修抽样和跨难度探索；
- 达到最低有效证据量后才切换到其他状态。

最低证据量必须按题型和目标配置，并在评估报告中说明。

## 7. 不确定性

`confidence` 综合：

- 有效证据数量；
- 证据权重；
- 证据时间跨度；
- 不同题型覆盖；
- 正负证据一致性；
- 题目校准质量；
- 教师确认比例。

出现以下情况时降低置信度：

- 只有一道题；
- 所有证据来自同一题型或同一模板；
- 最近表现与历史明显冲突；
- 题目映射刚修改；
- 大量答案处于复核中；
- 题目难度或区分度未知；
- 系统发生依赖降级。

推荐引擎必须同时考虑掌握度和置信度，不能把低置信度低分直接解释为薄弱点。

## 8. 常见错误状态

错误模式由稳定规则、评分 signal 或教师确认产生，例如：

数学：

- `DISTRIBUTION_MISSED_SECOND_TERM`；
- `NEGATIVE_SIGN_DROPPED`；
- `LIKE_TERMS_WRONG_VARIABLE`；
- `EQUATION_OPERATION_ONE_SIDE_ONLY`。

英语：

- `PAST_TENSE_MISUSE`；
- `SUBJECT_VERB_AGREEMENT`；
- `ARTICLE_OMISSION`；
- `PLURAL_FORM_ERROR`；
- `READING_CAUSE_EFFECT_MISSED`。

模型不得从一条自由文本模型解释中直接创建错误标签。错误模式需要：

- 稳定 `misconception_code`；
- 来源事件；
- 证据次数；
- 首次/最近出现时间；
- 置信度；
- 教师确认或解除状态。

## 9. 教师覆盖

教师可以：

- 将状态改为 `insufficient_evidence`；
- 标记某些事件不适合作为证据；
- 确认或解除错误模式；
- 冻结某目标的自动更新；
- 恢复自动更新；
- 关闭某学生或班级的推荐。

覆盖要求：

- 必须填写稳定原因码；
- 可选填写有限长度备注；
- 保存原状态、目标状态、操作者和时间；
- 不删除底层事件；
- 支持撤销；
- 后续重放必须尊重仍有效的覆盖记录。

## 10. 推荐系统接口边界

Student Model 输出状态，不直接返回具体题目。

推荐服务消费：

- 课程目标状态；
- 置信度；
- 先修关系；
- 遗忘风险；
- 错误模式；
- 教师范围配置。

推荐服务再从已发布题库选择题目。Student Model 不得调用 Generator Provider。

题库不足时只输出去标识化 content demand：

```json
{
  "objective_revision_id": "...",
  "question_type": "M2",
  "difficulty_band": "intermediate",
  "misconception_code": "NEGATIVE_SIGN_DROPPED",
  "required_count": 5
}
```

## 11. 评估

### 11.1 离线模型指标

- 校准误差；
- Brier score 或等价概率指标；
- 下一次作答预测，仅作为辅助；
- 状态稳定性；
- 单事件异常跳变率；
- 低证据误标掌握率；
- 教师覆盖率和覆盖原因；
- 目标/年级/题型分层表现。

### 11.2 教学效果指标

- 前测—后测学习增益；
- 延迟保持；
- 迁移题表现；
- 连续失败率；
- 提示依赖变化；
- 教师推荐接受率；
- 学生完成率和主动退出率。

模型不得仅凭下一题预测准确率进入生产。

## 12. 公平性

评估至少比较：

- 不同年级；
- 不同课程 profile；
- 英语与数学；
- 不同题型；
- 冷启动与高证据学生；
- 不同设备/网络条件下的证据缺失风险。

人口属性默认不进入模型。若为了公平性审计使用受保护属性，必须：

- 独立授权；
- 与在线推断隔离；
- 最小化访问；
- 只输出聚合结果；
- 明确保留期限。

重点检查不同群体获得的题目难度、探索机会、提示机会和课程覆盖是否存在系统性差异。

## 13. 隐私与安全

- 状态数据按租户隔离；
- 个人详情只向授权教师和管理员开放；
- 学生只看自己的解释性状态；
- 班级报告默认聚合；
- 导出个人明细需要额外权限和审计；
- 知识画像不得进入普通日志；
- 删除、同意撤回和处理限制必须停止状态更新与推荐；
- 外部 Generator 不接收个人画像；
- 模型配置、重放和教师覆盖均写入审计。

## 14. 监控

生产监控包括：

- Learning Event 积压；
- 状态更新时间延迟；
- 重放失败率；
- 低证据掌握标记率；
- 状态跳变率；
- 教师覆盖率；
- 推荐连续失败率；
- 不同版本输出差异；
- 数据删除遗漏；
- 跨租户访问拒绝。

出现异常时可以：

- 暂停状态更新；
- 保留事件继续入库；
- 关闭推荐；
- 回滚到上一个模型版本；
- 从固定事件水位重建状态。

## 15. 版本与回滚

每个状态快照保存：

- `model_id`；
- `model_version`；
- 配置版本；
- 证据游标；
- 课程目标 revision；
- 映射 revision；
- 生成时间。

升级流程：

```text
固定事件水位
→ 新版本离线重放
→ 与现网状态比较
→ 教师抽样审核
→ 公平性与学习指标检查
→ 影子运行
→ 小比例切换
→ 全量切换
```

回滚不改写事件，只切换读取的状态版本。旧快照按保留政策保存。

## 16. 已知限制

v1 可能受到以下限制：

- 题目—目标映射质量不足；
- 低频目标证据稀疏；
- 题目难度尚未充分校准；
- 教师评分差异；
- 提示和响应时间数据不完整；
- 学生在非平台场景中的学习无法观测；
- 短期情绪、设备或网络问题可能影响表现；
- 英语 E3/E4 需要教师确认，状态更新会延迟。

页面和 API 需要向使用者说明这些限制。

## 17. 发布门禁

Student Model v1 进入教师影子模式前：

1. Learning Event 契约与重放测试通过；
2. 题目—目标映射审核流程上线；
3. 低证据误标掌握率达到门槛；
4. 教师可以查看证据并覆盖状态；
5. 删除、同意撤回和跨租户测试通过；
6. 监控和 Kill Switch 可用；
7. 隐私影响评估获批。

进入有限学生自适应试点前还必须：

1. 教师影子阶段完成；
2. 推荐只使用已发布题目；
3. 连续失败保护和探索比例测试通过；
4. 学习增益与体验没有明显退化；
5. 推荐不影响正式成绩；
6. 学生和监护人说明、退出和人工救济流程可用。
