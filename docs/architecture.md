# 第一阶段架构

## 目标

优先跑通“布置—作答—批改—复核—发布—订正”闭环，而不是建设完整 LMS。

## 边界

- `apps/web`：学生与教师角色界面。首期共用一个 Nuxt 应用，路由和权限隔离。
- `apps/api`：身份、班级、题库、作业、提交、成绩、复核、订正与审计。
- `services/grader`：纯批改能力；不直接管理学生身份。
- PostgreSQL：业务数据和版本记录。
- Redis：缓存、幂等、任务队列与短期状态。

## 关键原则

1. 批改服务只接收去标识化题目规则和答案。
2. 题目、规则、答案和结果全部版本化。
3. 批改器异常进入复核，不默认给零分。
4. 数学表达式不解析未经清洗的字符串；只接收白名单 AST。
5. 英语语义信号不能单独把答案判为正确。
6. 成绩发布与批改执行分离，教师可在发布前抽查或复核。

## 初始部署

试点阶段使用 Docker Compose。达到多校、多环境或高并发需求后，再评估 Kubernetes、独立任务 Worker 和模型服务扩缩容。
