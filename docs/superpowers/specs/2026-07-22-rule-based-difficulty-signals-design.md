# 规则型难度信号设计

**日期：** 2026-07-22

**范围：** GitHub #40 的目标难度、估计难度、偏差和主要特征记录

## 根因

验证器已检查候选题声明难度是否在课程目标范围内，也会对已配置的年级复杂度阈值发出警告；但 `GenerationValidationRun.feature_summary_json` 没有保存一个可复核的难度估计，无法分析声明难度与题目结构是否一致。

## 决策

新增纯规则 `rule-based-difficulty-v1`，只使用已安全可得的结构特征：题干词汇单元、最长句单元、M1/M2 的最大数值尺度和 M2 运算节点数，以及题型基线。它输出 0–1 的估计值、候选声明的目标难度、偏差、课程允许范围和脱敏的特征/贡献项。

该信号写入每次验证运行的 `feature_summary_json.difficulty_signal`，不增加数据库迁移。它是解释性审计数据而非阻断依据：现有声明难度范围门禁不变，算法不调用模型、不读取 Prompt 正文、不使用自由字符串执行或新的外部依赖。

为避免重复和指标漂移，把已有年级复杂度警告使用的观测指标提取为同一安全辅助函数；M2 只使用已由 Grader 规范化并验证的 AST。

## 验收

- 每个正常验证运行都持久化版本、目标、估计、偏差、范围和稳定特征。
- M1/M2 的数值或运算特征可解释地影响估计；文本特征同样记录。
- 无效声明难度或无法安全取得 M2 AST 时以 `null` / 缺失原因表示，而不伪造数值或改变已有 blocked 结果。
- 运行摘要和 API payload 不含题干、规则原文、教师约束或模型输出正文。
