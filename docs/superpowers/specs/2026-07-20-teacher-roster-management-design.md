# 教师班级与学生名册管理设计

## 目标

让教师能够在教师端创建并管理自己负责的班级，并以单个录入或 CSV 导入方式维护学生名册；不授予教师全局管理员权限。

## 现状与根因

教师页 `apps/web/app/pages/teacher/index.vue` 是静态占位页面。创建班级和导入学生仅存在于 `/v1/admin/classes` 与 `/v1/admin/students/import`，并要求 `admin` 角色，因此教师账号既没有可见入口，也不能调用这些接口。

## 权限模型

- 教师创建班级时，系统自动创建 `ClassTeacher` 关联。
- 教师只可读取、创建和更新自己关联班级的名册。
- 教师不能访问其他教师的班级、学生名册、审计日志或全局数据。
- 管理员既有 `/v1/admin/*` 能力不变。
- 学生名册导入继续沿用现有监护人同意校验、原子事务与审计事件。

## 后端接口

新增独立的教师路由，而不放宽管理员路由：

- `GET /v1/teacher/classes`：列出当前教师负责的班级。
- `POST /v1/teacher/classes`：创建班级并自动关联当前教师。
- `POST /v1/teacher/classes/{class_id}/students`：录入单个学生。
- `POST /v1/teacher/classes/{class_id}/students/import`：导入该班的 CSV 名册。

服务层复用既有名册解析、监护人同意和审计逻辑；在写入前验证班级归属，避免通过请求参数跨班级写入。

## 教师端

教师首页改为可操作工作台：

1. 展示“我的班级”列表及每班学生数。
2. 提供创建班级表单（班级代码、名称）。
3. 在班级卡片中提供“添加学生”和“导入 CSV”。
4. 在接口拒绝访问、CSV 校验失败或写入失败时，显示可理解的错误信息。

## 验收与测试

- 教师可创建班级，且创建后自动成为该班教师。
- 教师可录入学生、导入有效名册，现有同意规则仍生效。
- 非归属教师访问或写入该班返回资源不存在或拒绝访问，且不泄露班级信息。
- 管理员现有接口和流程保持可用。
- 前端测试覆盖建班、学生录入/导入入口及失败提示；后端测试覆盖授权隔离和名册写入。
