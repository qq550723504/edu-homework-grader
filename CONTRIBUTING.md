# 贡献指南

## 分支与提交

- 从 `main` 创建短生命周期分支。
- 分支命名：`feat/...`、`fix/...`、`docs/...`、`chore/...`。
- 提交信息建议采用 Conventional Commits，例如 `feat(grader): add numeric tolerance grading`。
- 通过 Pull Request 合并；业务逻辑变更必须包含测试。

## 完成定义

一个任务只有在以下条件满足时才算完成：

1. 验收条件全部通过；
2. 新增或修改逻辑有自动化测试；
3. 关键评分决策返回可解释证据；
4. 不记录学生答案、令牌或身份信息到普通应用日志；
5. 文档、环境变量和接口契约同步更新。
